# Plan: Add Refresh Token Support to flow-space-ui

## Context
The backend (`flow-space-web-api`) now returns `accessToken` + `refreshToken` from `/sign-in` and supports `POST /refresh` for token rotation. The UI needs to consume these new fields and automatically refresh expired access tokens before signing the user out.

## Current Auth Flow
1. `auth.tsx` calls `POST /sign-in` → stores response in `localStorage('@userAuthData')` as `{ token, login, role }`
2. `use-auth-http-request.tsx` attaches `Authorization: Bearer ${userAuthData.token}` to every request
3. On 401 → immediately calls `signOut()` (no retry)

## Changes

### 1. Update `AuthUserModel` (`src/models/auth-user-model.tsx`)
- Rename `token` → `accessToken`
- Add `refreshToken: string`

### 2. Update `auth.tsx` (`src/contexts/auth.tsx`)
- `signIn`: store both `accessToken` and `refreshToken` from API response
- `signOut`: update to use `accessToken` (was `token`)
- Add `refreshAccessToken()` function: calls `POST /refresh` with `refreshToken`, updates localStorage, returns new `accessToken`
- Export `refreshAccessToken` from context

### 3. Update `AuthContextModel` (`src/models/auth-context.tsx`)
- Add `refreshAccessToken: () => Promise<string | null>` to the type

### 4. Update `use-auth-http-request.tsx` (`src/contexts/app-data/use-auth-http-request.tsx`)
- On 401 response: attempt `refreshAccessToken()` first
- If refresh succeeds → retry the original request with the new token
- If refresh fails → sign out (existing behavior)

### 5. Add route (`src/constants/app-api-routes.ts`)
- Add `accountRefresh: '/refresh'`

### 6. Update `signOut` in `auth.tsx`
- Change `userAuthData.token` → `userAuthData.accessToken`
