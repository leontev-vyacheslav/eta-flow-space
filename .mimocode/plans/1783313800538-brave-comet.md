# Plan: Add helmet security headers with CSP exception for /static

## Context

The API has no security headers. Add `helmet` with default headers, but set a permissive CSP only for the `/static` prefix (serves flow diagrams and schemas). Other routes get the default strict CSP.

## Step 1 — Install

```bash
npm install helmet
```

## Step 2 — Update `src/main.ts`

```ts
import helmet from 'helmet';

async function bootstrap() {
    const app = await NestFactory.create<NestExpressApplication>(AppModule, { ... });

    // Default helmet (strict CSP + all security headers)
    app.use(helmet());

    // Permissive CSP for /static assets (flow diagrams, schemas)
    app.use('/static', helmet({
        contentSecurityPolicy: false,          // disable CSP for /static
        crossOriginEmbedderPolicy: false,      // SVGs may need this off
    }));

    // ... rest unchanged
}
```

`app.use('/static', helmet({...}))` applies only to requests starting with `/static`. Other routes keep the default helmet headers (CSP, X-Frame-Options, HSTS, etc.).
