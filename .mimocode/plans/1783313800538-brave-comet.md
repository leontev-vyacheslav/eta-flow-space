# Plan: Fix timing attack on SHA-256 password comparison

## Context
The user implemented bcrypt migration and throttling. During review, we identified a timing attack vulnerability in the legacy SHA-256 password comparison path at `src/auth/auth.controller.ts:43`. The `!==` operator short-circuits on the first differing character, leaking hash information via response timing.

## Change

**File:** `src/auth/auth.controller.ts`

1. Update import on line 4:
   ```ts
   // Before
   import { createHash } from 'crypto';
   // After
   import { createHash, timingSafeEqual } from 'crypto';
   ```

2. Replace the comparison on lines 42-44:
   ```ts
   // Before
   const hashedPassword = createHash('sha256').update(signIn.password).digest('base64');
   if (hashedPassword !== user.password) {

   // After
   const hashedPassword = createHash('sha256').update(signIn.password).digest('base64');
   const hashedBuf = Buffer.from(hashedPassword);
   const storedBuf = Buffer.from(user.password);
   if (hashedBuf.length !== storedBuf.length || !timingSafeEqual(hashedBuf, storedBuf)) {
   ```

The length check is required because `timingSafeEqual` throws on mismatched lengths, and the comparison itself must be constant-time so you can't leak the length.
